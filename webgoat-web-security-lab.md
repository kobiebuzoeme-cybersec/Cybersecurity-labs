# OWASP WebGoat Web Application Security Lab

## Objective
Learn how common web application vulnerabilities occur and how they can be exploited and mitigated.

## Tool Used
- OWASP WebGoat
- Kali Linux
- Web Browser

## Lab Environment
Attacker Machine: Kali Linux  
Target Application: OWASP WebGoat  
Platform: Localhost

## Topics Explored
- SQL Injection
- Broken Authentication
- Cross-Site Scripting (XSS)
- Access Control vulnerabilities

## Methodology

1. Launch WebGoat application
2. Access training modules
3. Perform vulnerability exploitation exercises
4. Understand how each vulnerability works
5. Learn mitigation techniques

## Security Lesson
Web application vulnerabilities often occur due to poor input validation and insecure authentication mechanisms. Secure coding practices and proper validation help prevent these attacks
